## 아키텍처

### 트리거

```
python -m jobs.analyze 005930   # KR
python -m jobs.analyze AAPL     # US
```

시장 자동 감지: 숫자 6자리 → KR, 영문 티커 → US.

### 데이터 플로우

```
jobs/analyze.py
      │
      ├─ core/fundamentals.py  → 재무 지표 + 분기 실적 추이
      │       KR: DART API (dart-fss 라이브러리)
      │       US: yfinance
      │
      ├─ core/technicals.py    → 기술 지표 + 수급
      │       가격 이력: pykrx (KR) / yfinance (US)
      │       수급: pykrx (KR 전용)
      │
      ├─ core/news_fetcher.py  → 뉴스 헤드라인 (재사용)
      │
      ├─ core/reporter.py      → Claude 종합 의견 (함수 추가)
      │       섹터 평균 PER 감안 지시 포함
      │       단기 / 중장기 관점 분리 출력
      │
      └─ core/notifier.py      → 텔레그램 전송 (send_photo 추가)
              차트 이미지 (sendPhoto) + 텍스트 리포트 (sendMessage)
```

### 신규 파일

| 파일 | 역할 |
|------|------|
| `core/fundamentals.py` | 재무 데이터 수집 (DART / yfinance) |
| `core/technicals.py` | 기술 지표 + 수급 계산 |
| `jobs/analyze.py` | CLI 오케스트레이터 |

### 기존 파일 수정

## Claude 리포트 (`generate_analysis_report()`)

### 프롬프트 지시사항

- 섹터 평균 PER를 감안하여 현재 밸류에이션 수준 평가
- **단기 관점**: 기술 지표·수급·거래량 흐름 기반 진입 타이밍 의견
- **중장기 관점**: 재무 건전성·성장성·차트 구조(주봉 추세 + 피보나치) 종합
- 매수/매도 결정은 사용자가 최종 판단 (제안만)
- 텔레그램 Markdown 형식 출력

### 출력 형태 예시


## 에러 처리

graceful degradation — 단계별 실패 시 해당 섹션만 생략하고 계속 진행:

| 실패 상황 | 처리 |
|----------|------|
| DART API 키 미설정 | 재무 섹션 "조회 실패" 표시 후 계속 |
| 가격 이력 조회 실패 | 기술 지표 + 차트 전체 스킵 |
| 수급 조회 실패 (KR) | 수급 섹션만 생략 |
| 차트 생성 실패 | 텍스트 리포트만 전송 |
| Claude API 실패 | 수치 데이터만 그대로 전송 |
| 유효하지 않은 티커 | 즉시 에러 메시지 출력 후 종료 |
